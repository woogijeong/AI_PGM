# 파이썬 패키지 로드맵 정리


## 1. 큰 그림: 목적별 4가지 분야

| 분야 | 핵심 패키지 | 한 줄 요약 |
|---|---|---|
| 게임 개발 | Pygame | 그래픽/사운드/이벤트 루프로 2D 게임 제작 |
| 데이터 시각화 | Matplotlib, Seaborn, Plotly, Dash | 정적 그래프 → 통계 시각화 → 인터랙티브 → 웹 대시보드 |
| 웹 개발 | Django, Flask, FastAPI | 올인원 vs 미니멀 vs 고성능 비동기 |
| 자동화/크롤링 | Requests, Selenium | API 통신 vs 브라우저 상호작용 |

공통 기반: **Pandas(데이터 가공)**, **Numpy(수치 연산)** — 위 모든 분야의 밑바탕이 되는 패키지

---

## 2. 데이터 시각화 패키지 (4단계 비교)

| 패키지 | 특징 | 추천 상황 |
|---|---|---|
| **Matplotlib** | 모든 시각화의 기초. 세밀한 조절 가능하나 코드가 길고 디자인이 투박함 | 논문, 학술 보고서 |
| **Seaborn** | Matplotlib 기반, Pandas와 강하게 연동. 한 줄로 세련된 통계 그래프 | 통계 분석, EDA, 블로그 |
| **Plotly** | 줌/호버 등 인터랙티브 그래프, HTML로 바로 공유 가능 | 발표, 웹 리포트 |
| **Dash** | Plotly 기반, 파이썬만으로 웹 대시보드 구축 | 상호작용 데이터 앱 |

> **학습 순서**: Matplotlib(기초) → Seaborn(생산성) → Plotly/Dash(발표·대시보드)

## 3. 웹 프레임워크 3종 비교

| 항목 | Django | Flask | FastAPI |
|---|---|---|---|
| 철학 | 풀스택 (배터리 포함) | 마이크로 (BYOE) | 현대적 API (비동기) |
| 등장 | 2005년, 20년 검증 | 2010년 | 최신 |
| 학습 곡선 | 가파름 | 낮음 | 중간 (타입 힌트 필요) |
| 성능 | 양호 (WSGI) | 보통 (WSGI) | 최고 (ASGI 비동기) |
| 보안 | 내장형 포괄적 방어 | 수동 설정 | OAuth 2.0 등 현대 표준 |
| 대표 사례 | 인스타그램, 스포티파이 | 스타트업 MVP | - |
| 이럴 때 선택 | 대규모 상용 서비스 | 소규모 실험/빠른 시작 | 고성능 API 서버 |

## 4. 자동화(RPA) 패키지

| 패키지 | 용도 |
|---|---|
| **Requests** | API 통신, 속도 중심의 데이터 크롤링 |
| **Selenium** | 로그인 등 브라우저 상호작용이 필요한 동적 자동화 |

> 문서에 따르면 RPA는 단순 업무 자동화를 넘어, **AI 학습용 고품질 데이터셋을 확보하는 전처리 단계**로도 의미가 있습니다.

---

## 5. 의사결정 체크리스트 (원문 요약)

- 성능·비동기가 최우선 → **FastAPI**
- 올인원 검증된 솔루션 필요 → **Django**
- 결과를 예쁘고 빠르게 한 줄로 → **Seaborn**
- 사용자가 직접 그래프 조작 → **Plotly**
- 게임 만들며 구조적 사고 훈련 → **Pygame**
- 데이터 수집/자동화 → **Requests & Selenium**
